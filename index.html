GIT - Version Control

Setup

GIT Repository erstellen und initiieren
  • Rechtsklick auf Zielordner> GIT Bash

Zuerst wollen wir mit die Grundstruktur von GIT anlegen. Dies erstell einen Unterordner mit Konfigurationsdateien etc.
git init

Danach wollen wir die dateien hinzufügen die mit in die Versionskontrolle mit eingehen sollen.
git add .

Bevor wir die dateien abspeichern, überprüfen wir welche dateien für den Commit bereit stehen:
git status

Bevor wir die dateien abspeichern, überprüfen wir welche dateien für den Commit bereit stehen:
git commit -m "<CommitText>"

GIT Befehle

Dies sind die vorerst wichtigsten Befehle zusammengefasst

  • Erstellung/Initiierung von Repos.
    Mit gitbash im Hauptordner
    - git init
  • Lokale Versionsverwaltung
    git status zeigt den Status der STAGE und dessen Dateien
    git add [dateiname] fügt Änderungen zur STAGE hinzu [WD]>[STAGE]
    git commit speichert den aktuellen STAGE als COMMIT, also als Version. [STAGE]>[COMMIT]
    git log zeigt alle commits
    ... -n [zahl] zeigt eine bestimmte Anzahl von logs
    ... -grep="[Text]" durchsucht alle commits nach welchen die den angegebenen Text beinhalten
  • Branches
    git branch zeigt alle aktiven Branches
  • Remote Befehle (Git-Hub)
    git remote zeigt remote Repos.
    ... add origin [URL] fügt das Repo. auf der angegebenen Adresse als remote Repo. hinzu.
    ... -vzeigt alle adressen die für "fetch" und "pull" verwendet werden.
    git push [remote repo.] [local branch]

  • git clone
GIT Referenz für alle Befehle unter
git --help
Oder für spezifische Befehle mit
git <Befehl> --help
in die Konsole eingeben
